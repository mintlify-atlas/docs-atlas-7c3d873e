This page covers important information about privacy, data handling, and legal disclaimers for the Zoho Sprints Commit Helper Chrome extension.

## Privacy Statement

<Info>
**Your privacy is paramount.** This extension is designed with privacy-first principles.
</Info>

### What Data Is Accessed

The extension accesses:

1. **Your Own Tasks Only**
   - Only fetches tasks assigned to you (filtered by your Owner ID)
   - Only accesses tasks in the Done lane
   - Only reads task names, task IDs, and completion dates

2. **Authentication Information**
   - CSRF token from your browser cookies (for authenticated requests)
   - Session ID from page context or sessionStorage
   - Source identifier from page context or sessionStorage

3. **Configuration Settings**
   - Team ID, Project ID, Owner ID, Done Lane ID, Sprint ID
   - Stored locally in Chrome's storage API

### What Data Is NOT Collected

<Accordion title="No External Data Transmission">
  **We do NOT:**
  - Send any data to external servers
  - Transmit your tasks or configurations anywhere
  - Upload anything to the cloud
  - Report usage analytics
  - Track your activity

  **Technical Proof:**
  The extension only makes requests to:
  ```
  https://sprints.zoho.in/zsapi/*
  ```

  No other domains are contacted. Check the `manifest.json` file:
  ```json
  {
    "host_permissions": ["https://sprints.zoho.in/*"]
  }
  ```
</Accordion>

<Accordion title="No Data Storage Outside Your Browser">
  All data stays **100% local** in your browser:

  - Configuration settings: Stored in `chrome.storage.local`
  - Fetched tasks: Only in memory (RAM) while popup is open
  - Clipboard data: Only when you explicitly click "Copy"

  **Nothing persists to disk except your settings**, and those never leave your computer.
</Accordion>

<Accordion title="No Third-Party Services">
  - No analytics services (Google Analytics, Mixpanel, etc.)
  - No crash reporting (Sentry, Bugsnag, etc.)
  - No CDN resources (all code is local)
  - No external JavaScript libraries loaded at runtime
</Accordion>

<Accordion title="No Access to Other Websites">
  The extension **only** has permissions for:
  ```
  https://sprints.zoho.in/*
  ```

  It cannot access:
  - Other websites you visit
  - Your browsing history
  - Cookies from other domains
  - Data from other extensions
</Accordion>

### Chrome Permissions Explained

The extension requests these permissions (from `manifest.json`):

```json
{
  "permissions": ["activeTab", "scripting", "storage"],
  "host_permissions": ["https://sprints.zoho.in/*"]
}
```

**Why Each Permission Is Needed:**

| Permission | Purpose | Privacy Impact |
|------------|---------|----------------|
| `activeTab` | Check if you're on sprints.zoho.in | Only accesses current tab URL |
| `scripting` | Inject content script to access your session | Only runs on sprints.zoho.in |
| `storage` | Save your configuration settings | Stored locally, never synced |
| `host_permissions` | Make API calls to Zoho Sprints | Only contacts sprints.zoho.in |

<Note>
No "sensitive" permissions are requested:
- No `<all_urls>` (access to all websites)
- No `tabs` (access to all tab information)
- No `cookies` global permission
- No `webRequest` (intercept network traffic)
- No `storage.sync` (sync to Google account)
</Note>

## Security Considerations

<Accordion title="Open Source = Auditable">
  **Advantage:** You can review the source code yourself.

  All code is available for inspection:
  - `manifest.json` - Permissions and configuration
  - `popup.js` - UI logic and task handling
  - `content.js` - Authentication and API calls
  - `popup.html` - User interface
  - `styles.css` - Styling

  **What to Check:**
  - No external network requests (except to sprints.zoho.in)
  - No data exfiltration
  - No malicious code
  - Proper authentication handling
</Accordion>

<Accordion title="Authentication Security">
  **How Authentication Works:**
  1. You log into Zoho Sprints normally
  2. Extension reads your CSRF token from cookies
  3. Extension extracts session ID from page context
  4. Extension includes these in API requests

  **Security Notes:**
  - Your credentials are never touched by the extension
  - Tokens are never stored persistently
  - All requests use `credentials: 'same-origin'`
  - CSRF protection is maintained

  **From content.js:**
  ```javascript
  const csrfToken = document.cookie
    .split('; ')
    .find((row) => row.startsWith('CSRF_TOKEN='))
    ?.split('=')[1] || '';
  
  // ... later ...
  headers['x-zcsrf-token'] = `zscp=${csrfToken}`;
  ```
</Accordion>

<Accordion title="No Secret Storage">
  The extension does NOT store:
  - Passwords
  - API keys
  - Tokens (beyond current session)
  - Sensitive personal information

  **Only Stored:**
  - Team ID (public identifier)
  - Project ID (public identifier)
  - Owner ID (your user ID)
  - Done Lane ID (public identifier)
  - Sprint ID (public identifier)

  These IDs are not secret - they're visible in URLs and network requests on Zoho Sprints.
</Accordion>
